# ICH E6(R3) Annex 2 — GCP: 分散型・プラグマティック試験とRWD

## 概要

ICH E6(R3)の附属書2（Annex 2）。2024年11月6日に Step 2 として承認・公開コンサルテーション中。Annex 1（通常GCP要件）の補足として、**分散型要素（Decentralised Elements）**・**プラグマティック要素（Pragmatic Elements）**・**リアルワールドデータ（RWD）**を組み込んだ臨床試験に特有のGCP考慮事項を提供する。

> [!stale] ステータス確認
> 本文書は2024年11月時点でStep 2（公開コンサルテーション）。Step 4（確定版）への移行状況は要確認。

## 適用対象

| 要素 | 定義 |
|------|------|
| **分散型要素** | 治験依頼者サイト外での試験関連活動（在宅訪問、地域医療センター、モバイルユニット、DHTによるリモートデータ収集） |
| **プラグマティック要素** | 臨床診療の側面を試験に組み込む（簡略プロトコル、効率化されたデータ収集） |
| **RWD** | 臨床試験外で収集されたデータ（EHR、レジストリ、クレームデータ）。エンドポイント確認や外部対照として利用 |
| **DHT** | ウェアラブル、モバイルアプリ等のデジタルヘルステクノロジー |

## 主要要件・ポイント

### 1. IRB/IEC

プライバシー・機密性・データセキュリティへの特別な注意が必要。

### 2. 治験責任医師（Investigator）

| トピック | 要件 |
|---------|------|
| **リモートIC** | 適切な場合はリモート取得可。**参加者の本人確認**が必要（規制要件に従い） |
| **IC材料** | どのデータを収集・利用・共有するかを明示（在宅訪問時のアドレス情報等） |
| **IP管理** | 在宅・地域薬局等への直接配送可（規制要件に従い）。適切な監督の維持が必要 |
| **治験責任医師の監督** | リスクおよびデータの重要性に**比例**した監督レベル |
| **安全性モニタリング** | 複数ソース（在宅看護・DHT・EHR・リモート受診）からの安全情報を統合してレビュー |

### 3. スポンサー

| トピック | 要件 |
|---------|------|
| **早期エンゲージメント** | 患者・患者団体、医療従事者、規制当局への早期関与 |
| **プロトコル記載** | 設計要素・データソースの十分な記載と使用の正当化 |
| **RWDの同意** | 適用規制要件に従いRWD利用への同意・許可を取得 |
| **RWDの信頼性** | 信頼性（accuracy・completeness・traceability）と関連性（必要データ要素の利用可能性）の確保 |
| **リモートデータ収集** | サイバーセキュリティ・データプライバシーへの特別な注意 |
| **プライバシー** | DHT・RWD利用時のデータ漏洩リスクへの対策 |
| **スポンサー監督** | 多数のデータソース・サービスプロバイダーへの適切な監督体制 |
| **安全性管理** | 多様なソースからの安全情報を収集・治験責任医師に提供（actionable manner） |

### RWD信頼性の2軸

| 軸 | 定義 |
|----|------|
| **Reliability（信頼性）** | Accuracy（正確性）+ Completeness（完全性）+ Traceability（追跡可能性） |
| **Relevance（関連性）** | 必要なデータ要素（曝露・アウトカム・共変量）の利用可能性 |

## 実務上のTips

> [!key-insight] 規制当局との早期相談が必須
> 分散型要素・RWDを含む試験は**規制当局への早期相談**（Pre-IND/Pre-CTA/PMDA事前面談）が強く推奨される。ガイダンスの解釈が各国で異なる可能性がある。

> [!key-insight] RWDの規制アクセス権を契約に明記
> スポンサーが管理しないRWDソース（EHR、レジストリ等）については、**規制当局が原資料にアクセスできる契約**をデータ保有者と締結することが必要。

> [!key-insight] ICH E19との組み合わせ
> 後期試験・市販後試験での選択的安全性データ収集についてはICH E19も参照するよう明示されている。

## 注意点・よくある落とし穴

> [!gap] 各国規制要件との整合
> Annex 2は「applicable regulatory requirements（適用規制要件）に従い」という留保が多用される。各国（FDA/EMA/PMDA）の個別ガイダンスとの整合確認が必要。

> [!gap] Step 2ドラフトの実務適用
> 本文書はStep 2（公開コンサルテーション）であり確定版ではない。実務では現行の確定済みガイドライン（E6(R2)等）を基礎としつつ参考として使用すること。

> [!gap] RWDと欠測データ
> RWDでは、患者が医療機関を変わる等によるデータ欠損や、中間事象（治療中断・追加治療）の捕捉困難が生じやすい。ICH E9(R1)のEstimand framework との整合が必要。
